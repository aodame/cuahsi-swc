<a name="basic-shell-commands" class="anchor" href="#basic-shell-commands"><span class="octicon octicon-link"></span></a>Basic Shell Commands</h1>

<hr><h2>
<a name="1-shell-basics" class="anchor" href="#1-shell-basics"><span class="octicon octicon-link"></span></a>1. Shell Basics:</h2>

<table>
<thead><tr>
<th>Command</th>
<th>Definition</th>
</tr></thead>
<tbody>
<tr>
<td><code>.</code></td>
<td>a single period refers to the current directory</td>
</tr>
<tr>
<td><code>..</code></td>
<td>a double period refers to the directory immediately above the current directory</td>
</tr>
<tr>
<td><code>~</code></td>
<td>refers to your home directory. <em>Note:</em> this command does NOT work on Windows machines (Mac and Linux are okay)</td>
</tr>
<tr>
<td><code>cd ./dirname</code></td>
<td>changes the current directory to the directory <code>dirname</code>
</td>
</tr>
<tr>
<td><code>ls -F</code></td>
<td>tells you what files and directories are in the current directory</td>
</tr>
<tr>
<td><code>pwd</code></td>
<td>tells you what directory you are in (<code>pwd</code> stands for *p*rint *w*orking *d*irectory)</td>
</tr>
<tr>
<td><code>history</code></td>
<td>lists previous commands you have entered. `history</td>
</tr>
<tr>
<td>
<code>man</code> <em>cmd</em>
</td>
<td>displays the *man*ual page for a command.</td>
</tr>
</tbody>
</table><h2>
<a name="2-creating-things" class="anchor" href="#2-creating-things"><span class="octicon octicon-link"></span></a>2. Creating Things:</h2>

<h3>
<a name="a-how-to-create-new-files-and-directories" class="anchor" href="#a-how-to-create-new-files-and-directories"><span class="octicon octicon-link"></span></a>a) How to create new files and directories..</h3>

<table>
<thead><tr>
<th>Command</th>
<th>Definition</th>
</tr></thead>
<tbody>
<tr>
<td><code>mkdir ./dirname</code></td>
<td>makes a new directory called dirname below the current directory. <em>Note:</em> Windows users will need to use <code>\</code> instead of <code>/</code> for the path separator</td>
</tr>
<tr>
<td><code>nano filename</code></td>
<td>if <code>filename</code> does not exist, <code>nano</code> creates it and opens the <code>nano</code> text editor. If the file exists, <code>nano</code> opens it. <em>Note:</em> <em>(i)</em> You can use a different text editor if you like.  In gnome Linux, <code>gedit</code> works really well too. <em>(ii)</em> <code>nano</code> (or <code>gedit</code>) create text files. It doesn't matter what the file extension is (or if there is one)</td>
</tr>
</tbody>
</table><h3>
<a name="b-how-to-delete-files-and-directories" class="anchor" href="#b-how-to-delete-files-and-directories"><span class="octicon octicon-link"></span></a>b) How to delete files and directories...</h3>

<h4>
<a name="remember-that-deleting-is-forever-there-is-no-going-back" class="anchor" href="#remember-that-deleting-is-forever-there-is-no-going-back"><span class="octicon octicon-link"></span></a><em>Remember that deleting is forever. There is NO going back</em>
</h4>

<table>
<thead><tr>
<th>Command</th>
<th>Definition</th>
</tr></thead>
<tbody>
<tr>
<td><code>rm ./filename</code></td>
<td>deletes a file called <code>filename</code> from the current directory</td>
</tr>
<tr>
<td><code>rmdir ./dirname</code></td>
<td>deletes the directory <code>dirname</code> from the current directory. <em>Note:</em> <code>dirname</code> must be empty for <code>rmdir</code> to run.</td>
</tr>
</tbody>
</table><h3>
<a name="c-how-to-copy-and-rename-files-and-directories" class="anchor" href="#c-how-to-copy-and-rename-files-and-directories"><span class="octicon octicon-link"></span></a>c) How to copy and rename files and directories...</h3>

<table>
<thead><tr>
<th>Command</th>
<th>Definition</th>
</tr></thead>
<tbody>
<tr>
<td><code>mv tmp/filename .</code></td>
<td>moves the file <code>filename</code> from the directory <code>tmp</code> to the current directory. <em>Note:</em> <em>(i)</em> the original <code>filename</code> in <code>tmp</code> is deleted. <em>(ii)</em> <code>mv</code> can also be used to rename files (e.g., <code>mv filename newname</code>
</td>
</tr>
<tr>
<td><code>cp tmp/filename .</code></td>
<td>copies the file <code>filename</code> from the directory <code>tmp</code> to the current directory. <em>Note:</em> <em>(i)</em> the original file is still there</td>
</tr>
</tbody>
</table><h2>
<a name="3-pipes-and-filters" class="anchor" href="#3-pipes-and-filters"><span class="octicon octicon-link"></span></a>3. Pipes and Filters</h2>

<h3>
<a name="a-how-to-use-wildcards-to-match-filenames" class="anchor" href="#a-how-to-use-wildcards-to-match-filenames"><span class="octicon octicon-link"></span></a>a) How to use wildcards to match filenames...</h3>

<p>Wildcards are a shell feature that makes the command line much more powerful than any GUI file managers. 
Wildcards are particularly useful when you are looking for directories, files, or file content that can
vary along a given dimension.  These wildcards can be used with any command that accepts file names or 
text strings as arguments.</p>

<h4>
<a name="table-of-commonly-used-wildcards" class="anchor" href="#table-of-commonly-used-wildcards"><span class="octicon octicon-link"></span></a>Table of commonly used wildcards</h4>

<table>
<thead><tr>
<th>Wildcard</th>
<th>Matches</th>
</tr></thead>
<tbody>
<tr>
<td><code>*</code></td>
<td>zero or more characters</td>
</tr>
<tr>
<td><code>?</code></td>
<td>exactly one character</td>
</tr>
<tr>
<td><code>[abcde]</code></td>
<td>exactly one of the characters listed</td>
</tr>
<tr>
<td><code>[a-e]</code></td>
<td>exactly one character in the given range</td>
</tr>
<tr>
<td><code>[!abcde]</code></td>
<td>any character not listed</td>
</tr>
<tr>
<td><code>[!a-e]</code></td>
<td>any character that is not in the given range</td>
</tr>
<tr>
<td><code>{software,carpentry}</code></td>
<td>exactly one entire word from the options given</td>
</tr>
</tbody>
</table><p>See the cheatsheet on regular expressions for more "wildcard" shortcuts.</p>

<h3>
<a name="b-how-to-redirect-to-a-file-and-get-input-from-a-file-" class="anchor" href="#b-how-to-redirect-to-a-file-and-get-input-from-a-file-"><span class="octicon octicon-link"></span></a>b) How to redirect to a file and get input from a file ...</h3>

<p>Redirection operators can be used to redirect the ouput from a program from the display screen to a file where it is saved (or many other places too, like your printer or to another program where it can be used as input).</p>

<table>
<thead><tr>
<th>Command</th>
<th>Description</th>
</tr></thead>
<tbody>
<tr>
<td><code>&gt;</code></td>
<td>write <code>stdout</code> to a new file; overwrites any file with that name (e.g., <code>ls *.md &gt; mardkownfiles.txt</code>)</td>
</tr>
<tr>
<td><code>&gt;&gt;</code></td>
<td>append <code>stdout</code> to a previously existing file; if the file does not exist, it is created (e.g., <code>ls *.md &gt;&gt; markdownfiles.txt</code>)</td>
</tr>
<tr>
<td><code>&lt;</code></td>
<td>assigns the information in a file to a variable, loop, etc (e.g., <code>n &lt; markdownfiles.md</code>)</td>
</tr>
</tbody>
</table><h4>
<a name="b1-how-to-use-the-output-of-one-command-as-the-input-to-another-with-a-pipe" class="anchor" href="#b1-how-to-use-the-output-of-one-command-as-the-input-to-another-with-a-pipe"><span class="octicon octicon-link"></span></a>b.1) How to use the output of one command as the input to another with a pipe...</h4>

<p>A special kind of redirection is called a pipe and is denoted by <code>|</code>. </p>

<table>
<thead><tr>
<th>Command</th>
<th>Description</th>
</tr></thead>
<tbody><tr>
<td>|</td>
<td>Output from one command line program can be used as input to another one (e.g. ls *.md | head gives you the first 5 <code>*.md</code> files in your directory)</td>
</tr></tbody>
</table><h5>
<a name="example" class="anchor" href="#example"><span class="octicon octicon-link"></span></a>Example:</h5>

<pre><code>ls *.md | head | sed -i `s/markdown/software/g`
</code></pre>

<p>changes all the instances of the word <code>markdown</code> to <code>software</code> in the first 5 <code>*.md</code> files in your current directory.</p>

<h2>
<a name="4-how-to-repeat-operations-using-a-loop" class="anchor" href="#4-how-to-repeat-operations-using-a-loop"><span class="octicon octicon-link"></span></a>4. How to repeat operations using a loop...</h2>

<p>Loops assign a value in a list or counter to a variable that takes on a different value each time through the loop.
There are 2 primary kinds of loops: <code>for</code> loops and <code>while</code> loops.</p>

<h3>
<a name="a-for-loop" class="anchor" href="#a-for-loop"><span class="octicon octicon-link"></span></a>a) For loop</h3>

<p>For loops loop through variables in a list</p>

<pre><code>for varname in list
do
    command 1
    command 2
done
</code></pre>

<p>where,</p>

<ul>
<li> <code>for</code>, <code>in</code>, <code>do</code>, and <code>done</code> are keywords</li>
<li> <code>list</code> contains a list of values separated by spaces. e.g. <code>list</code> can be replaced by <code>1 2 3 4 5 6</code> or by <code>Bob Mary Sue Greg</code>. <code>list</code> can also be a variable:</li>
</ul><h2></h2>

<pre><code>list[0]=Sam
list[1]=Lynne
list[2]=Dhavide
list[3]=Trevor
.
.
.
list[n]=Mark
</code></pre>

<p>which is referenced in the loop by:</p>

<pre><code>for varname in ${list[@]}
do
    command 1
    command 2
done
</code></pre>

<p><em>Note:</em> Bash is zero indexed, so counting always starts at <code>0</code>, not <code>1</code>.</p>

<h3>
<a name="b-while-loop" class="anchor" href="#b-while-loop"><span class="octicon octicon-link"></span></a>b) While Loop</h3>

<p>While loops loop through the commands until a condition is met. For example</p>

<pre><code>COUNTER=0
while [ ${COUNTER} -lt 10 ]; do
    command 1
    command 2
    COUNTER=`expr ${COUNTER} + 1` 
done
</code></pre>

<p>continues the loop as long as the value in the variable COUNTER is less than 10 (incremented by 1 on each iteration of the loop).</p>

<ul>
<li> <code>while</code>, <code>do</code>, and <code>done</code> are keywords</li>
</ul><h4>
<a name="b1-commonly-used-conditional-operators" class="anchor" href="#b1-commonly-used-conditional-operators"><span class="octicon octicon-link"></span></a>b.1) Commonly used conditional operators</h4>

<table>
<thead><tr>
<th>Operator</th>
<th>Definition</th>
</tr></thead>
<tbody>
<tr>
<td><code>-eq</code></td>
<td>is equal to</td>
</tr>
<tr>
<td><code>-ne</code></td>
<td>is not equal to</td>
</tr>
<tr>
<td><code>-gt</code></td>
<td>greater than</td>
</tr>
<tr>
<td><code>-ge</code></td>
<td>greater than or equal to</td>
</tr>
<tr>
<td><code>-lt</code></td>
<td>less than</td>
</tr>
<tr>
<td><code>-le</code></td>
<td>less than or equal to</td>
</tr>
</tbody>
</table><p>Use <code>man bash</code> or <code>man test</code> to learn about other operators you can use.</p>

<h3>
<a name="a-how-to-select-lines-matching-patterns-in-text-files" class="anchor" href="#a-how-to-select-lines-matching-patterns-in-text-files"><span class="octicon octicon-link"></span></a>a) How to select lines matching patterns in text files...</h3>

<p>To find information within files, you use a command called <code>grep</code>.</p>

<th>Example command</th>
<th>Description</th>
</tr></thead>

</tr></tbody>
</table><h4>
<a name="a1-commonly-used-grep-options" class="anchor" href="#a1-commonly-used-grep-options"><span class="octicon octicon-link"></span></a>a.1) Commonly used <code>grep</code> options</h4>

<th></th>
<th>
<code>grep</code> options</th>
</tr></thead>

<tr>
<td><code>-i</code></td>
<td>makes matching case-insensitive</td>
</tr>

<td><code>-n</code></td>

</tr>
<tr>
<td><code>-v</code></td>
<td>shows lines that do not match the pattern (inverts the match)</td>
</tr>
<tr>
<td><code>-w</code></td>

</tr>
</tbody>
</table><h3>

</tr></tbody>
</table><h4>
<a name="b1-commonly-used-find-options" class="anchor" href="#b1-commonly-used-find-options"><span class="octicon octicon-link"></span></a>b.1) Commonly used <code>find</code> options</h4>
